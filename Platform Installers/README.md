# 📱 Platform Installers

## 📊 Data Storage Information

### ✅ **Local Storage Details:**
- **Storage Type**: Browser localStorage (persistent)
- **Data Location**: 
  - **Mac**: `~/Library/Application Support/Google/Chrome/Default/Local Storage`
  - **Windows**: `%LOCALAPPDATA%\Google\Chrome\User Data\Default\Local Storage`  
  - **Linux**: `~/.config/google-chrome/Default/Local Storage`
- **Backup**: Built-in backup/restore system available
- **Offline**: Works completely without internet
- **Cross-Device**: Use backup files to sync between computers

### 💾 **Data Persistence:**
- All transactions automatically saved
- Employee records stored locally
- Settings and preferences maintained
- No data sent to external servers
- Complete privacy and security

## 📂 Platform-Specific Installers

### 🍎 **Mac (macOS 10.12+)**
📁 **Folder**: `Mac/`
📄 **File**: `Esthetics-Auto-CashBook-1.0.0-Mac.dmg` (77KB)

**Installation:**
1. Double-click `.dmg` file
2. Drag app to Applications folder
3. Launch from Applications or Launchpad
4. Data stored in browser's localStorage

### 🖥️ **Windows (Windows 7+)**  
📁 **Folder**: `Windows/`
📄 **File**: `Esthetics-Auto-CashBook-1.0.0-Windows.zip` (47KB)

**Installation:**
1. Extract ZIP to desired location
2. Double-click `CashBook.vbs` (silent launch)
3. Or run `CashBook.bat` (with console)
4. Data stored in browser's localStorage

### 🐧 **Linux (Ubuntu/Debian)**
📁 **Folder**: `Linux/`  
📄 **File**: `esthetics-auto-cashbook_1.0.0_linux.tar.gz` (48KB)

**Installation:**
1. Extract: `tar -xzf esthetics-auto-cashbook_1.0.0_linux.tar.gz`
2. Run included install script or manual setup
3. Launch from applications menu
4. Data stored in browser's localStorage

## 🔄 **Data Migration Between Devices**

### Method 1: Backup/Restore
1. **Export**: Settings → Create Backup → Download JSON file
2. **Transfer**: Copy backup file to new device
3. **Import**: Settings → Import Data → Select backup file

### Method 2: Manual Export
1. **Export Excel**: Reports → Export to Excel  
2. **Transfer file** to new device
3. **Re-enter data** in new installation
